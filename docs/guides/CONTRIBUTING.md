## ğŸ“‹ **Contribution Guidelines**

### **Pull Request Process**
1. **Before You Start**: Check existing issues and PRs to avoid duplicates
2. **Small PRs**: Keep changes focused and atomic when possible
3. **Description**: Clearly describe what your PR does and why
4. **Testing**: Ensure all existing functionality still works
5. **Documentation**: Update relevant docs for new features
6. **Review**: Be responsive to feedback and suggestions

### **Commit Messages**
Use conventional commit format:
```
type: brief description

Longer explanation if needed

- type: feat, fix, docs, style, refactor, test, chore
- Use present tense ("add feature" not "added feature")
- Use imperative mood ("move cursor to..." not "moves cursor to...")
- Limit first line to 72 characters
```

Examples:
```
feat: add Claude Sonnet support to creative workflow
fix: resolve emoji encoding issues in generated games
docs: update quick start guide with new CLI commands
```
